20180602 Shell Style Guide

Google Shell Style Guide 中有对我有指导意义的：

  1. 如果需要使用 ${PIPESTATUS} 之外其他的数组，考虑使用Python
  2. 如果代码超过100行，考虑使用Python
  3. 不要用 .sh 作为可执行文件扩展名，库文件必须用 .sh

  4. 使用 STDERR 输出错误信息；推荐使用一个函数同时输出其他状态信息

  5. 写文件注释

  6. 写方法注释，包括功能、使用的和改变的全局变量、接受参数、除了最后命令的返回值
  7. 缩进2个空格
  8. 每行不超过80字符；否则使用 here document 或 embedded newline
  9. 循环中 ; do ; then 放在 while for if 同一行
  10. 变量使用时，单个字符除非必要，不使用大括号，如 $1 $?；其他都使用
  11. Never quote literal integers
  12. Use an explicit path when doing wildcard expansion of filenames: ./* instead of *
  13. 方法命名：小写，下划线分隔，需要括号，可不写开头的 function
  14. 变量命名同上
  15. 常数及环境变量，大写，下划线分隔，定义在文件开头
  16. 文件名小写，可用下划线分隔
  17. 只读变量使用 readonly 或 declare -r

  18. 局部变量使用 local；如果值是从其他命令获得，声明和赋值必须分两行（因为 local 不会传递exit code)

  19. 函数位置在常数下，之间不要有可执行代码

  20. 使用main函数，将程序代码放在其中；main函数放在所有函数之后，调用时使用 main "$@"
  21. 尽量使用 Builtin Commands，而不是外部命令，如 sed 等
