1. Scala不支持像Java中用集合调用另一个集合的创建方法，比如 `Set<String> set = new HashSet<>(someList)`，但是可以使用空集合与已有集合相加，如 `val set = HashSet() ++ someList`
2. Scala中方法要获取域的值时，是通过方法获得的，所以就会被被override和多态影响，这点和Java不同；所以分析起来也简单一些，可以理解为子类的域覆盖了父类的同名域，任何操作都是直接操作的子类的域；而Java则同时存在父类的域与子类的同名域
```java
class F1 {
    private int a = 1;

    public int getA(){
        return a;
    }
    public int getF1A() {
        return a;
    }

}
class F2 extends F1 {
    private int a = 2;
    public int getA(){
        return a;
    }
    public int getF2A(){
        return a;
    }
}
public class JavaTest {
    public static void main(String[] args) {
        F2 f2 = new F2();
        System.out.println(f2.getF1A()); // 1
        System.out.println(f2.getF2A()); // 2
        System.out.println(f2.getA()); // 2

        F1 f1 = new F2();
        System.out.println(f1.getF1A()); // 1
        System.out.println(f1.getA()); // 2 多态
    }
}
```

  ```scala
object ScalaTest extends App {
  class F1{
    val a = 1
    def getA = a
    def getF1A = a // 实际调用a()方法获取a的值，所以会被子类重载
  }

  class F2 extends F1{
    override  val a = 2
    override def getA = a
    def getF2A = a
  }

  val f2 =new F2
  println(f2.getF1A) // 2 因为F1的a()被F2的a()重载
  println(f2.getF2A) // 2
  println(f2.getA) // 2

  val f1: F1 = new F2
  println(f1.getF1A) // 2 因为F1的a()被F2的a()重载
  println(f1.getA) // 2
}
  ```
