1. 在测试 partial function 的时候写出了如下代码
```scala
  def divide(i: Int): PartialFunction[Int, Int] = { 
    case x if x != 0 => 42 / x
  }
  println(divide(2)) // result: <function1>
```
  希望输出结果是21，但却是一个function，迷惑了好久，后来发现问题在于定义中的 `i`，这其实是对函数定义不清造成的。 
  a. 上面代码其实是一个currying function，所以要调用的话需要两个参数，如 `divide(1)(2)`，当然第一个参数在这里没有意义
  b. 如果写成 `def divide(): ...` 的话，就需要这样调用
  ```scala
  divide()(2) // 21
  divide.apply(2) // 21
  divide(2) // 错误，因为第一个括号不接受参数
  ```
  当然不建议这种定义方式，因为没有副作用就不用写括号
  c. 如果写成 `def divide: ...` 的话，就需要这样调用 `divide(2)`；实质上 `divide` 本身返回了一个函数，相当于 `divide.apply(2)`
  
  
2. [方法调用](https://docs.scala-lang.org/style/method-invocation.html)
  a. Infix notation；适用于运算符，所有以字母开头的方法，不建议使用，如 `names mkString ","`，例外是比较像运算符且符合交换性的，如 `a max b`
  b. Postfix Notation；不建议使用
