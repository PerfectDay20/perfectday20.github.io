1. `getOrElse(default: => B)` 中的 `default` 可以直接添加 `return` 返回结果；但是假如传入一个带有 `return` 的函数对象则没有返回效果
```scala
  def makeFunner(word: String): String = {
    Option(word).getOrElse(return "I like fun!")
    s"$word is fun"
  }

  makeFunner("paint") // "paint is fun"
  makeFunner(null) // "I like fun!"

  def foo:String = "I like fun!"
  def makeFunner2(word:String):String = {
    Option(word).getOrElse(foo)
    s"$word is fun"
  }

  makeFunner2("paint") // "paint is fun"
  makeFunner2(null) // "null is fun"
```
2. 不要使用 `null`，使用 `Option` 可以更安全，更函数化；在使用 `Option` 的时候，不用 `get`，而用 `map` `flatMap` `for` 或 pattern matching

[Scala best practices: null values, Option, Some, and None](https://alvinalexander.com/scala/scala-null-values-option-uninitialized-variables)

3. Scala `for` 语句是通过编译成 `map` `flatMap` `withFilter` `foreach` 来实现的，所以拥有这些方法的类都可以用 `for`；`withFilter` 是 non-strict 的，也就是不生成中间集合；Java中的 `for` 是编译成 `Iterator` 或数组下标；Python中是 `__getitem__` `__iter__` `__next__`
4. `Option` 的 `map` 实现是通过对其中包含对象的处理，所以对其使用 `for` 的时候，只有有值的时候才有结果
```Scala
  @inline final def map[B](f: A => B): Option[B] =
    if (isEmpty) None else Some(f(this.get))
    
  for (i <- Some(1)) println(i) // print 1
  for (i <- None) println(i) // no action
```
