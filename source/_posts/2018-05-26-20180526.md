```scala
class Creature{
  val range: Int = 10
  val env: Array[Int] = new Array[Int](range)
}

class Ant extends Creature{
  override val range: Int = 2
}
```

当子类覆盖父类的val且该val用于初始化父类时，会因为初始化顺序的不同产生问题。
比如上面这个例子，目的是为了让Ant的env长度为2，但是初始化的顺序是：
1. 父类range设为10，生成getter和setter
2. 初始化env，调用range的getter，因为被子类覆盖，调用子类的range getter
3. 由于子类range未初始化，值为0，getter返回0，则父类的env长度初始化为0
4. 初始化子类range为2

这样在生成Ant实例时，env长度就变为了0
和下面的Java代码效果相同：
```java
class Creature{
    private int range = 10;
    private int[] env = new int[getRange()];

    public int getRange() {
        return range;
    }

    public int[] getEnv() {
        return env;
    }
}

class Ant extends Creature{
    private int range = 2;
    @Override
    public int getRange(){
        return range;
    }
}
```
简单来说就是父类初始化调用了被子类覆盖的方法，而该方法又使用了子类中还未初始化的变量。
解决方案：
1. final val，不允许覆盖
2. lazy val，但效率低
3. early definition syntax：`class Ant extends{override val range = 2} with Creature`（由于trait没有构造参数，所以该语法经常用于覆盖父trait中的域，如`class A extends {override val f = 1} with B`）
