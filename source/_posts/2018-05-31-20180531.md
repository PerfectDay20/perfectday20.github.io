1. par.foreach 是无序的，但是par.reduce/fold等结果是有序的
2. 改变并行度：
```scala
  val p = 1.to(10).par
  p.tasksupport = new ForkJoinTaskSupport(new ForkJoinPool(10))
```
3. Future改变默认线程池：
```scala
  val pool = Executors.newFixedThreadPool(10)
  implicit val ec = ExecutionContext.fromExecutor(pool)
```
4. `_` 不仅可以用在方法参数中，也可以用在定义函数变量上，如
```scala
  val f1 = (a: Int, b: Int) => a + b
  val f2 = (_: Int) + (_: Int)
```
5. `_` 放在方法名后面，将创建一个 partially applied function，此时 `_` 代表该方法的整个参数列表，如` println _ `；其实更普通的用法，或者说partially体现在：
```scala
  def f(a: Int, b: Int) = a + b
  val partialF = f(1, _:Int)
```
当整个参数列表都省略了，就可以只在后面添加 `_`
在需要一个函数变量的位置，可以省略掉最后的 `_`，比如 `Array(1,2,3).map(println)`
6. Scala的闭包方法会根据外面引入值的变化而变化，但Java中不允许修改引入的值，如
```scala
  var a = 1
  def f(b:Int) = a+b
  println(f(1)) // 2
  a = 2
  println(f(1)) // 3
```
7. Scala编译器会对尾递归进行优化，不会生成多个调用栈，但Java不会优化
