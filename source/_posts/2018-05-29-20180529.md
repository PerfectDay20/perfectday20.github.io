1. implicit conversion 格式进化
```scala
// 原来需要写一个class，然后加上一个implicit def作为转换，如
  case class Foo(i:Int)
  implicit def toFoo(i:Int) = Foo(i)
// 后来进化到直接在类前机上implicit，则编译器自动生成转换方法，如
  implicit case class Foo(i:Int)
```
2. implicit parameter 格式进化
```scala
// 1. 原来需命名implicit参数，并在body中使用该变量名，如
  def foo[T](a:T)(implicit ordering: Ordering[T])={
    ordering.gt(???)
  }
// 2. 后来可以不用变量名，使用implicitly方法，如
  def foo[T](a:T)(implicit ordering: Ordering[T])={
    implicitly[Ordering[T]].gt(???)
  }
// 3. 最后进化到使用context bounds，不用添加implicit参数列表，如
  def foo[T:Ordering](a:T)={
    implicitly[Ordering[T]].gt(???)
  }
```
3. implicit conversion 和 implicit parameter 可以叠加使用，比如
```scala
  def small[T:Ordering](t1: T, t2:T) = {
    import Ordered._
    if (t1 < t2) t1 else t2
  }
// 这里又一个隐式变量: Ordering[T]，然后Ordered trait中有一个orderingToOrdered方法接受这个隐式变量，将t1转换为Ordered对象，就拥有了'<'这个方法
```
4. 隐式转换只能被编译器使用一次，但是int -> double这种数字类型转换和子类当作父类使用这种转换其实是属于 Weak Conformance，所以和隐式转换可以叠加使用，比如
```scala
  class A(val i: Int)
  case class B(j: Int) extends A(j)
  case class C(k: Int) {
    def foo(c: C) = 1
  }

  implicit def aToC(a: A) = C(a.i)

  val b = B(1)
  val c = C(2)
  // 这里b是B类型，当时可以作为父类A使用，被通过aToC隐式转换为C
  c.foo(b)
  b.foo(c)

```

3. 放在方法的参数前，导入隐式变量
4. 放在类前，生成隐式转换方法
5. 放在方法前，隐式转换方法
