1. Spark SQL 中用Scala对Row中Column进行修改，可以先写一个udf，然后用udf.apply。（20180607: 其实并不用udf，见代码部分）
```scala
  def transform(a:A, b:B):T = ???
  ds.withColumn("name", udf(transform _).apply($"colA", $"colB")
  // 或者
  // val transform = (a:A, b:B) => ???
  // ds.withColumn("name", udf(transform).apply($"colA", $"colB")

  // 20180607 更新
  ds.withColumn("name", udf(transform)($"colA", $"colB") // wrong
  // 注意上面由于udf是一个currying方法，第二个参数表是implicit的，所以第二个括号不能是Column对象，必须使用apply方法
  // 要对Column修改，除了写通用的类型转换方法，比如 String=>String，可以直接写 Column=>Column 方法，这样就免去了转udf的过程，如：
  def foo(col:Column): Column = col.doSomeThing
  ds.withColumn("name", foo('colA))
```

3. Scala Symbol 用于表示这个String并不是单纯的字符串，而是有其它含义，比如变量名、方法名等，在引入spark.implicits._后，Symbol可以被隐式转换为Column，`'colA` 比 `$"colA"` 更加清晰；当然另一个好处就是String被interned，equal比较更快
