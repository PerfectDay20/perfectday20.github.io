1. Java的 `File.deleteOnExit()` 在JVM异常退出时不会执行，所以不能根据是否存在进程生成的pid文件判断是否正在运行，还得直接查询进程
2. [查询进程方法](https://stackoverflow.com/questions/3043978/how-to-check-if-a-process-id-pid-exists)
```bash
if ps -p "${pid}" > /dev/null; then
  echo "${pid} is running"
fi
```
3. 读tab分隔的文件，可以直接创建schema，然后利用 `DataFrameReader` 的 `csv`方法；这样省去了自己切割及创建bean类及转换对象等操作
```scala
  val schema = StructType(Seq(StructField("name", StringType), StructField("age", IntegerType)))
  val ds = spark.read.option("sep", "\t")
    .schema(schema)
    .option("mode", "DROPMALFORMED")
    .option("ignoreLeadingWhiteSpace","true")
    .option("ignoreTrailingWhiteSpace","true")
    .csv("path")
    
  // schema 也可以由 case class 获得
  case class Person(name:String, age:Int)
  val schema = Encoders.product[Person].schema
  
  // 直接 as 是不行的，因为列名默认是 [_c0, _c1]
  val ds = spark.read.option("sep", "\t").csv("path").as[Person]
```
4. 写Spark的时候经常需要对 `Dataset` 做一些复杂的转换，这时利用 `Dataset.transform(someOperation)` 可以更好地复用代码，更易读，更易测试
5. Dataset transform方法，Column方法，UDF 正好对应了 Dataset=>Dataset, Column=>Column, T=>U 这三种类型；总之还是尽量使用Spark native API，少用UDF
